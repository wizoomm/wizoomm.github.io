2. Цель работы#

Изучить основные возможности Vue 3:

  • Связывание данных (когда данные на странице меняются автоматически)
  • События (когда пользователь что-то делает кликает, вводит текст)
  • Computed свойства (вычисляемые значения, которые пересчитываются автоматически)
  • Watch (отслеживание изменений в данных)
  • Формы и работа с ними
  • Условное отображение элементов (показывать/скрывать в зависимости от условий)
  • Вывод списков задач, их сортировка и фильтрация
  • Компоненты, передача данных между ними, события и слоты
  • Маршрутизация (переход между страницами) через Vue Router

Научиться работать с сервером (REST API), изучить основы создания сервера на Python Flask, научиться упаковывать приложение в Docker контейнеры и правильно оформить отчет по проекту.

3. Что было сделано#

3.1. Фронтенд#

Компоненты (отдельные части интерфейса)#

  1. AppHeader.vue — шапка сайта с меню

    • Ссылки на главную страницу, список задач и создание задачи
    • Используется Vue Router для перехода между страницами

  2. AppFooter.vue — нижняя часть сайта

  3. TaskList.vue — компонент, который показывает список задач

    • Получает данные о задачах и настройках фильтров
    • Использует computed свойства для фильтрации и сортировки списка
    • Отправляет события при изменении статуса, редактировании или удалении задачи

  4. TaskItem.vue — компонент одной задачи

    • Получает данные об одной задаче
    • Показывает информацию о задаче
    • Отправляет события при действиях пользователя
    • Использует слоты для дополнительной информации

  5. TaskForm.vue — форма для создания или редактирования задачи

    • Использует v-model для связи полей формы с данными
    • Проверяет, что все нужные поля заполнены
    • Отслеживает изменения данных через watch
    • Использует компонент LayoutCard со слотами

  6. LayoutCard.vue — компонент-карточка с рамкой

    • Обычный слот для основного содержимого
    • Именованные слоты для шапки и подвала карточки
    • Показывает шапку и подвал только если они есть

Страницы#

  1. Home.vue (/) — главная страница

    • Приветственный экран с большим заголовком
    • Кнопка для перехода к списку задач
    • Переход осуществляется через router-link

  2. TasksList.vue (/tasks) — страница со списком задач

    • Показывает все задачи через v-for (цикл)
    • Можно фильтровать по статусу (все / выполненные / невыполненные)
    • Можно сортировать по дате добавления или по алфавиту
    • Показывает сообщение, если задач нет
    • Загружает данные при открытии страницы (mounted)

  3. TaskCreate.vue (/tasks/new) — страница создания задачи

    • Форма с полями: заголовок, описание, приоритет, категория, флажок “Важно”
    • Проверка, что все обязательные поля заполнены
    • Отправка данных на сервер через API

  4. TaskEdit.vue (/tasks/:id/edit) — страница редактирования задачи

    • Та же форма, что и для создания, но с уже заполненными данными
    • Использует watch, чтобы загрузить данные задачи и заполнить форму

  5. NotFound.vue (/:pathMatch(.*)*) — страница 404

    • Показывается, когда пользователь переходит по несуществующей ссылке
    • Кнопка для возврата на главную

Переходы между страницами (маршрутизация)#

  • Все переходы реализованы через Vue Router
  • Использованы именованные маршруты для удобства
  • Навигация через this.$router.push()
  • Динамические маршруты для редактирования (например, /tasks/1/edit, /tasks/2/edit)

Computed свойства и Watch#

  • TaskList.vue: computed свойства filteredTasks и filteredAndSortedTasks для фильтрации и сортировки задач
  • TaskForm.vue: watch для отслеживания загрузки данных задачи и заполнения формы при редактировании

3.2. Бэкенд (серверная часть на Python Flask)#

Методы для работы с данными (CRUD)#

  1. GET /api/tasks — получить все задачи
  2. GET /api/tasks/<id> — получить одну задачу по её номеру
  3. POST /api/tasks — создать новую задачу
  4. PUT /api/tasks/<id> — обновить существующую задачу
  5. DELETE /api/tasks/<id> — удалить задачу

Хранение данных#

  • Используется база данных SQLite (простая файловая база данных)
  • Все данные хранятся в файле tasks.db
  • Подключение через встроенную библиотеку Python sqlite3
  • Таблица создается автоматически при первом запуске приложения

Структура таблицы tasks#

CREATE TABLE IF NOT EXISTS tasks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    description TEXT,
    status TEXT DEFAULT 'pending',
    priority TEXT DEFAULT 'medium',
    category TEXT DEFAULT 'work',
    important INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)

Таблица содержит поля:

  • id — номер задачи (увеличивается автоматически)
  • title — заголовок задачи (обязательное поле)
  • description — описание задачи
  • status — статус задачи (по умолчанию ‘pending’ - в работе)
  • priority — приоритет (по умолчанию ‘medium’)
  • category — категория (по умолчанию ‘work’)
  • important — важная ли задача (0 или 1)
  • created_at — когда создана (автоматически)
  • updated_at — когда обновлена (автоматически)

3.3. Docker#

  • Frontend Dockerfile: файл для упаковки фронтенда в контейнер (сборка в несколько этапов: сначала Node.js для сборки проекта, потом Nginx для отдачи готовых файлов)

5. Примеры кода#

5.1. Использование компонента TaskItem с событиями#

<TaskItem
  v-for="task in filteredAndSortedTasks"
  :key="task.id"
  :task="task"
  @toggle-status="handleToggleStatus"
  @edit="handleEdit"
  @delete="handleDelete"
/>

Этот код показывает, как используется компонент TaskItem. Для каждой задачи в отфильтрованном и отсортированном списке создается компонент TaskItem, которому передаются данные задачи. При действиях пользователя (изменение статуса, редактирование, удаление) вызываются соответствующие функции.

5.2. Computed свойства для фильтрации и сортировки#

computed: {
  filteredTasks() {
    if (this.statusFilter === 'all') {
      return this.tasks
    }
    return this.tasks.filter(task => task.status === this.statusFilter)
  },
  filteredAndSortedTasks() {
    const tasks = [...this.filteredTasks]
    if (this.sortBy === 'date') {
      return tasks.sort((a, b) => new Date(b.created_at) - new Date(a.created_at))
    } else if (this.sortBy === 'alphabet') {
      return tasks.sort((a, b) => a.title.localeCompare(b.title))
    }
    return tasks
  }
}

Это computed свойства, которые автоматически пересчитываются при изменении исходных данных. Первое свойство фильтрует задачи по статусу, второе сортирует отфильтрованные задачи.

5.4. Использование watch#

watch: {
  initialData: {
    immediate: true,
    handler(newVal) {
      if (newVal) {
        this.formData = {
          title: newVal.title || '',
          description: newVal.description || '',
          // ...
        }
      }
    }
  }
}

Watch отслеживает изменения данных. Когда меняется initialData, автоматически заполняется форма данными задачи.

5.5. Отправка запросов на сервер#

// services/api.js
export default {
  async getTasks() {
    const response = await axios.get('/api/tasks')
    return response.data
  },
  async createTask(taskData) {
    const response = await axios.post('/api/tasks', taskData)
    return response.data
  },
  // ...
}

Это функции для отправки запросов на сервер. getTasks получает все задачи, createTask создает новую задачу.

7. Как запустить проект#

7.1. Запуск через Docker Compose#

  1. Нужно установить Docker и Docker Compose

  2. Скачать проект из репозитория:

git clone https://github.com/wizoomm/TaskManager.git
cd TaskManager
  1. Запустить проект:
docker-compose up --build
  1. Открыть в браузере:
  • Фронтенд: http://localhost:3000
  • Бэкенд API: http://localhost:8000

База данных SQLite создастся сама в файле backend/tasks.db.

7.2. Запуск без Docker (для тестирования)#

Бэкенд:#

cd backend
pip install -r requirements.txt
python3 main.py

База данных SQLite создастся сама в файле tasks.db.

Фронтенд:#

cd frontend
npm install
npm run dev

В процессе выполнения этого задания я изучила много нового. На фронтенде я освоила Vue 3: научилась работать с Options API, поняла как данные автоматически обновляют интерфейс, изучила основные директивы (v-if, v-for, v-model, v-bind, v-on). Помимо этого, я изучила Docker: научилась создавать Dockerfile для фронтенда и бэкенда, поняла как работает сборка в несколько этапов, настроила Nginx для отдачи готового фронтенда.

В целом, я научилась правильно организовывать структуру проекта, разделять код на компоненты, страницы и сервисы, использовать роутер для навигации и централизованно работать с API.
