Отчет по заданию из темы 6

October 26, 2025 · 2 min · wizoomm

Используя LLM для объяснения, можно подробно разобрать выражение:

console.time('a');((y = 2 ** 14 , x = Date.now()) => { while (Date.now() - x < y) ;})();console.timeEnd('a');

Этот код запускает таймер с меткой ‘a’, затем выполняет немедленно вызываемое функциональное выражение (IIFE — Immediately Invoked Function Expression).
Внутри стрелочной функции задаются две локальные переменные со значениями по умолчанию:

  • y = 2 ** 14 — вычисляет 2 в 14-й степени, что равно 16384 миллисекунды (примерно 16,384 секунды);
  • x = Date.now() — получает текущее время в миллисекундах.

Далее выполняется цикл while (Date.now() - x < y) ;, который просто ожидает, пока разница между текущим временем и моментом старта x не превысит значение y. Это так называемое активное ожидание (busy-wait), при котором процессор постоянно проверяет условие, не выполняя других задач.
После завершения ожидания вызывается console.timeEnd('a'), который останавливает таймер и выводит прошедшее время. Результат в консоли примерно равен 16384 ms, то есть около 16 секунд.

Ответ на вопрос 1#

Переменная z нигде не объявлена и не используется в данном выражении.
Если попытаться обратиться к z, произойдёт ошибка:
ReferenceError: z is not defined.
Следовательно, переменная z не существует и не имеет значения.

Ответ на вопрос 2#

Переписываем выражение без IIFE.
IIFE — это функция, которая создаётся и сразу вызывается, чтобы изолировать переменные от внешнего контекста.
Аналогичный код без IIFE можно записать двумя способами:

Вариант 1 — линейно:

console.time('a');
const y = 2 ** 14;
const x = Date.now();
while (Date.now() - x < y) { /* ждём */ }
console.timeEnd('a');

Вариант 2 — с отдельной функцией:

function busyWait(ms) {
  const start = Date.now();
  while (Date.now() - start < ms) { /* пусто */ }
}

console.time('a');
busyWait(2 ** 14);
console.timeEnd('a');
